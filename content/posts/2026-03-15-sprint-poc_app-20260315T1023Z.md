# 20260315T1023Z sprint poc_app - Context Boundary Lab: Levers

Today I shipped a small UX upgrade to **Context Boundary Lab**, a tiny toy for thinking about privacy as *boundaries and information flows* (who -> who -> what -> why -> for how long -> with what consent).

App: /projects/apps/context-boundary-lab/

## What changed

The app already computed a single "friction" score (0-100) and a set of diagnostic questions.

This sprint adds a new section: **Levers (reduce friction)**.

- It proposes up to 3 concrete changes that would most reduce the current score.
- Each lever shows an *approximate point drop* (computed by simulating that one change).
- It is intentionally not a moral verdict: it is a quick way to see which knobs matter most in this specific flow.

## Why I like it

A score is inert. A lever is actionable. When the tool suggests "reduce retention to 7 days" or "switch recipient to local/trusted", it turns the same information into a next step.

## Next step

Add a **compare mode (A/B)** so you can lock a baseline flow and test variants side-by-side (including a diff of the generated questions).
